# Jobs-to-be-Done Canvas

> People do not buy products. They hire them to make progress in their lives.
> This canvas maps the job your users are trying to get done, the forces that drive or resist change, and the criteria for success.
> Fill this in based on evidence from interviews and research -- not assumptions.

## 1. Core Job Statement

> A job statement describes the progress someone is trying to make in a particular circumstance.
> Use this format: **When [trigger/situation], I want to [make progress / accomplish something], so I can [desired outcome].**

### Primary Job

> **When** [trigger or situation that creates the need]
> **I want to** [the progress the person is trying to make]
> **So I can** [the ultimate outcome or benefit they seek]

**Example:**
> **When** multiple vendor contracts come up for renewal in the same quarter,
> **I want to** quickly assess each vendor's performance and negotiate better terms,
> **So I can** reduce unnecessary spend and demonstrate procurement value to leadership.

## 2. Forces Diagram

> Four forces determine whether someone will switch from their current solution to a new one.
> For change to happen, Push + Pull must be stronger than Anxiety + Habit.

```
                    DEMAND GENERATION                    DEMAND REDUCTION
              (forces promoting change)            (forces opposing change)

    ┌─────────────────────────┐          ┌──────────────────────────────┐
    │                         │          │                              │
    │   PUSH of the Current   │          │   ANXIETY of the New        │
    │   Situation              │          │   Solution                  │
    │                         │          │                              │
    │   "This is broken and   │          │   "What if the new thing    │

    └─────────────────────────┘          └──────────────────────────────┘

    ┌─────────────────────────┐          ┌──────────────────────────────┐
    │                         │          │                              │
    │   PULL of the New       │          │   HABIT of the Present      │
    │   Solution              │          │                              │
    │                         │          │   "I know how this works.   │
    │   "That new thing looks │    vs    │    It's not great, but      │
```
